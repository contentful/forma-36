## Toolbar

---

### Purpose

The toolbar pattern provides global and consistent access to critical tools and contextually relevant apps within the platform. It addresses user needs for quick actions, enhances efficiency by minimizing navigation friction, and aligns closely with user workflows. By providing focused and immediate interactions, the toolbar pattern supports a streamlined editing experience, integrates seamlessly with different platform views, and reinforces the user's core jobs-to-be-done across various tasks.

---

### When to Use

Use the toolbar pattern when users frequently need quick, persistent access to tools and functionalities directly related to their primary tasks, enabling them to perform actions without leaving their current context.

#### Typical Use Cases

- Engaging agents for tasks like spinning up new experiences, viewing analytics insights, or performing other specialized tasks through a chat interface.
- Quick access to optimization tools relevant to content performance and management.
- Collaborative workflows that require persistent commenting and feedback mechanisms.
- Seamless integration and access to third-party apps directly within the editing context.
- Workflow management activities that benefit from immediate and persistent tool accessibility.

#### Consider These Alternatives
### Use a Different Pattern Instead

Consider other patterns when persistent access to tools is unnecessary or when providing extensive tool sets might overwhelm the user.

| Scenario                                                       | Alternative Pattern | Reason                                                                        |
|----------------------------------------------------------------|---------------------|-------------------------------------------------------------------------------|
| Simple actions within a limited context                        | Action Buttons      | A toolbar would add unnecessary complexity                                    |                 |
| Single task-focused tools without ongoing context              | Modal Dialog        | Ensures users remain focused without permanent interface elements             |
| One-time setup or configuration tasks                          | Wizard              | Provides step-by-step guidance without persistent toolbar complexity          |
| Primary actions that should always be prominently available    | Header Actions      | Ensures immediate visibility for critical actions                             |
| Essential information and actions visible at all times         | Persistent Sidebar  | Offers constant visibility within content layout                              |
| Long forms and extensive editing tasks                         | Dedicated Page      | Provides appropriate space and focus for extensive content manipulation       |

### Typical Interactions

#### Example Flow

1. User selects an icon in the toolbar.
2. The corresponding panel expands.
3. The user performs their required action within the panel (e.g., viewing insights, commenting, or task management).
4. Once completed, the user can collapse the panel via the close button in the panel header or by clicking the corresponding icon in the toolbar again.
5. Alternatively, the user can switch directly to another toolbar item as needed.
