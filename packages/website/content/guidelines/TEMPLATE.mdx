> * This template is intended for documenting patterns. Make sure you’ve identified the correct level of abstraction:
>    * Component – A reusable UI building block with a single, well-defined responsibility (for example, Button, Dialog, Text Input).
>    * Pattern – A reusable solution to a recurring user problem, typically composed of multiple components and interaction rules (for example, Toolbar, Empty State, Page Header).
>    * Template – A reusable page or screen structure that combines multiple patterns into a complete experience (for example, Dashboard, Editor, Settings page).
> * Focus on user goals and design rationale before implementation details.
> * Write guidance that is broadly applicable rather than feature-specific.
> * Use examples from the product to make the guidance concrete.
> * Link to existing documentation instead of duplicating it.
